# Apache Airflow проекты

## 1. `dag1` & `dag2` — создание DAG для определения последовательности загрузки/обновления таблиц и сборки витрины

**Проблема**: Данные из операционных систем (чеки, трафик, промо) загружались вручную, что приводило к задержкам и ошибкам в витрине.

**Решение**: Спроектировала DAG, который:
- Загружает справочники (`stores`, `promos`) через full-load
- Обрабатывает событийные таблицы (`coupons`, `traffic`) с преобразованием типов
- Загружает факт-таблицы (`bills_head`, `bills_item`) инкрементально через `delta_partition`
- Собирает итоговую витрину `mart_store_day`
- Включает retry-логику (1 попытка, задержка 5 мин) и алертинг при падении

**Стек**: Apache Airflow, PythonOperator, PostgresOperator, TaskGroup, Greenplum, Cron Scheduling

**Результат**:
- Полная автоматизация ежедневного обновления витрины
- Время выполнения пайплайна: ~8 минут
- При сбое — автоматический перезапуск, уведомление ответственного
- Шаблон легко масштабируется на новые источники

## 2. `dag3` — создание DAG для загрузки витрины duckdb в облачное хранилище s3 (фрагмент).

**Проблема**: Готовые витрины в DuckDB требовалось передавать в облачное хранилище для дальнейшей обработки в других системах.

**Решение**: Реализовала DAG для:
- Подключения к локальной DuckDB
- Выгрузки данных в Parquet/CSV
- Загрузки файлов в Yandex Object Storage (S3-compatible) через boto3/CLI
- Логирования процесса и обработки ошибок

**Стек**: Apache Airflow, DuckDB, Yandex Object Storage (S3), Python, Boto3

**Результат**:
- Автоматическая выгрузка витрин в облако по расписанию
- Готовый шаблон для интеграции с другими cloud-хранилищами
- Снижение ручного труда на 2–3 часа в неделю
