# Python проекты

## Проекты анализа данных:

### 1. `strlit_bi.py` — создание веб-приложения для визуализации данных с возможностью выгрузки в форматах csv, xlsx.

**Проблема**: Бизнес-пользователи часто запрашивали выгрузки по контрактам/лотам, что отвлекало аналитиков на рутинные задачи.

**Решение**: Разработала веб-приложение на Streamlit, которое позволяет:
- Загружать Excel-файлы с данными (контракты/лоты)
- Фильтровать по текстовым, числовым и дата-полям
- Выбирать отображаемые колонки
- Скачивать результат в CSV/XLSX с сохранением форматов (текстовые ID)

**Стек**: Python, Streamlit, Pandas, OpenPyXL

**Результат**:
- Сокращение ручных запросов на выгрузки на ~30%
- Пользователи самостоятельно формируют нужные срезы данных
- Код включает обработку ошибок и кэширование данных (`@st.cache_data`)

### 4. `tours_price_analysis.py` — анализ стоимости туров (очистка данных, визуализация, выводы).

**Проблема**: Необходимо было проанализировать динамику цен на туры от разных поставщиков для выявления конкурентных преимуществ.

**Решение**: Написала скрипт, который:
- Загружает CSV с данными о турах
- Проводит очистку: обработка пропусков, приведение типов
- Строит визуализации: распределение цен по странам, динамика во времени
- Экспортирует выводы в виде графиков (Seaborn/Matplotlib)

**Стек**: Python, Pandas, Seaborn, Matplotlib, EDA

**Результат**:
- Готовый шаблон для быстрого анализа новых датасетов
- Выявлены 3 страны с наибольшей волатильностью цен
- Код легко адаптируется под другие источники данных

### 5. `parsing_script.py` — пример простого парсинга поставщиков туров (использование requests + BeautifulSoup, извлечение данных и выгрузка в csv).

**Проблема**: Данные от некоторых поставщиков поступали только в виде HTML-страниц, без API или выгрузок.

**Решение**: Создала парсер на `requests + BeautifulSoup`, который:
- Обходит пагинацию (страницы 1–5)
- Извлекает поля: название тура, страна, цена, рейтинг
- Сохраняет результат в CSV с кодировкой UTF-8
- Добавляет `time.sleep()` для соблюдения politeness policy

**Стек**: Python, Requests, BeautifulSoup4, CSV, Web Scraping

**Результат**: 
- Автоматизация сбора данных с 1 источника (~100 записей/запуск)
- Готовый шаблон для подключения новых сайтов-поставщиков
- Код включает базовую обработку ошибок (можно расширить)
